## あなたのタスク

`$ARGUMENTS` をテーマに、ブログ記事の下書きを作成してください。

## 文体のルール

- ゆるく、個人的なトーンで書く（砕けた日本語でOK）
- 少し小説っぽい、文章に美しさのある表現を意識する
- 感情や情景の描写を丁寧に入れる
- 技術的な話題でも、体験談・心情を交えて書く
- 読んでいて「この人らしいな」と感じられるような文章にする
- 一人称は「私」

## 文体のサンプル

`content/blog/hello-world.md` を Read で読んで、このブログの文体・トーンを把握してから書いてください。

## 手順

1. `content/blog/hello-world.md` を読んで文体を把握する
2. 今日の日付を `date` コマンドで取得する（`date +%Y-%m-%d`）
3. タイトルからファイル名用のスラッグを考える（英数字とハイフンのみ、例: `typescript-generics`）
4. 以下のフォーマットで `content/blog/draft/日付-スラッグ.md` に下書きを書く

## ファイルフォーマット

```
---
title: "記事タイトル"
date: "YYYY-MM-DD"
description: "記事の概要（1〜2文）"
tags: ["タグ1", "タグ2"]
---

本文をここに書く
```

## 本文の構成ガイド

- 書き出しは読み手を引き込む一文から始める（説明から入らない）
- 見出し（##）は2〜4つ程度
- 1500〜2500字を目安に書く
- 最後は締めくくりの言葉で終わる（余韻を残す）
- 下書きなので、完璧でなくていい。書き出しと流れを作ることを優先する
